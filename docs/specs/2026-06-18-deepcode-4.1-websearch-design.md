# deepcode 4.1 WebSearch（双源 Bocha+Tavily）设计 spec

**日期：** 2026-06-18
**机制：** master roadmap 第 4 层 4.1。WebSearch 工具——闭合 deepcode「有 WebFetch 抓取、缺搜索」的环。
**对齐依据：** CC `tools/WebSearchTool`（schema/提示词/输出格式可移植；其 Anthropic 服务端 `web_search_20250305` server tool 不可移植，deepcode 改客户端直连第三方搜索 API）。Provider API 形态实查（Tavily 官方文档 + Bocha 4 份独立代码实现交叉核实）。
**TUI：** 否（纯工具 + 配置；接线同 WebFetch）。
**前置：** WebFetch 已有（`src/tools/webfetch.ts`，undici+ProxyAgent 模式）；用户提供 Bocha+Tavily key 已存 `~/.deepcode/settings.json`。

---

## 0. 关键事实

- **DeepSeek 无原生联网搜索**，CC 的 WebSearch 是 Anthropic 服务端 server tool（不可移植）。deepcode 走**客户端直连第三方搜索 API**。
- 用户钦定 **双源同时查 + 合并**（Bocha + Tavily 并行取并集去重，覆盖最全；代价=烧双份额度 + 延迟取最慢那家，用户已接受）。
- **本机必走代理**（127.0.0.1:8118）：对 Tavily/Bocha 的 HTTP 调用必须用 **undici + ProxyAgent**（镜像 webfetch.ts），否则连不通。
- **Provider 字段差异**（实查核实）：
  - **Tavily**：`POST https://api.tavily.com/search`，`Authorization: Bearer tvly-…`；body `{query, max_results, include_domains?, exclude_domains?}`；返回 `{results:[{title,url,content,score}]}`（content=snippet）；**原生域名过滤**。
  - **Bocha**：`POST https://api.bochaai.com/v1/web-search`，`Authorization: Bearer sk-…`；body `{query, count, freshness?}`；结果在 **`data.webPages.value[]`**，标题字段是 **`name`（非 title）**，`snippet` always；**无原生域名过滤**（仅 freshness）。caveat：freshness 用小写 `oneDay/oneWeek/oneMonth/oneYear/noLimit`；无公开限流。

## 1. 目标与范围

把 CC WebSearch 的工具形状忠实移植到 deepcode，搜索后端改双源（Bocha+Tavily）客户端直连。

### 做
- 工具 `WebSearch`（schema 从 CC 照搬：`query` + `allowed_domains?`/`blocked_domains?` 互斥）。
- provider 抽象 `src/webSearch.ts`：`WebSearchResult{title,url,snippet}` + `bochaSearch`/`tavilySearch`（各自 undici+ProxyAgent fetch + 字段归一化）。
- **双源并行**：对所有「配了 key」的 provider 用 `Promise.allSettled` 并发查 → 合并 → 按归一化 URL 去重 → 截顶 → 格式化。一家失败不影响另一家。
- 域名过滤跨源：Tavily 映射原生 include/exclude_domains；Bocha 客户端按域名过滤结果。
- 输出格式移植 CC（`Web search results for query: "…"` + 编号结果含 snippet + `REMINDER: Sources markdown 超链接` 尾）。
- `config.ts` `parseWebSearchConfig`（读 settings.webSearch 的 bocha/tavily key，env 可覆盖）。
- 接线 useChat/headless（同 WebFetch 站点）。

### 不做（YAGNI）
- provider 选源开关 / `/websearch` 命令（双源并查，无需选）。settings 里残留的 `provider` 字段忽略不用。
- 暴露 freshness/time_range/topic/search_depth 给模型（CC schema 也无；默认值即可）。
- Bocha `summary`（长 AI 摘要）——用 `snippet` 够且省 token。
- 抓取结果页全文（那是 WebFetch 的活；WebSearch 只给 title/url/snippet，模型需要全文再 WebFetch）。
- 结果排序融合算法（round-robin 交错即可，不做 score 归一化跨源排序）。

## 2. provider 抽象（`src/webSearch.ts`）

```ts
export interface WebSearchResult { title: string; url: string; snippet: string }

export interface WebSearchOpts {
  allowedDomains?: string[]
  blockedDomains?: string[]
  count?: number              // 每源请求条数；默认 5
  signal?: AbortSignal
}

/** undici + ProxyAgent fetch（镜像 webfetch.ts 的代理处理）。可注入便测。 */
export type FetchJson = (url: string, init: { method: string; headers: Record<string,string>; body: string; signal?: AbortSignal }) => Promise<any>

export async function bochaSearch(apiKey: string, query: string, opts: WebSearchOpts, fetchJson?: FetchJson): Promise<WebSearchResult[]>
export async function tavilySearch(apiKey: string, query: string, opts: WebSearchOpts, fetchJson?: FetchJson): Promise<WebSearchResult[]>
```

**默认 `fetchJson`**（`defaultFetchJson`，模块内）：用 `undici` fetch + `ProxyAgent`（proxy 来自 `https_proxy/HTTPS_PROXY/http_proxy/HTTP_PROXY` env，与 webfetch.ts 一致）→ POST → `res.json()`；非 2xx 抛 `Error(HTTP <status>)`。

**`bochaSearch`**：
- POST `https://api.bochaai.com/v1/web-search`，header `Authorization: Bearer <apiKey>` + `Content-Type: application/json`，body `{ query, count: opts.count ?? 5, freshness: 'noLimit' }`。
- 解析 `json.data?.webPages?.value ?? []` → map 每条 `{ title: v.name ?? '', url: v.url ?? '', snippet: v.snippet ?? '' }`，过滤 url 空的。
- Bocha 无原生域名过滤 → 调用方（makeWebSearchTool）对返回结果做客户端域名过滤。

**`tavilySearch`**：
- POST `https://api.tavily.com/search`，header `Authorization: Bearer <apiKey>` + `Content-Type: application/json`，body `{ query, max_results: opts.count ?? 5, ...(allowedDomains?.length ? {include_domains: allowedDomains} : {}), ...(blockedDomains?.length ? {exclude_domains: blockedDomains} : {}) }`。
- 解析 `json.results ?? []` → map `{ title: r.title ?? '', url: r.url ?? '', snippet: r.content ?? '' }`，过滤 url 空。

> 两函数都在自身 try 外抛错由调用方 allSettled 兜（单源失败不致命）。

## 3. 合并 + 去重（`src/webSearch.ts`）

```ts
export function normalizeUrl(u: string): string           // 小写 host + 去 fragment + 去尾部 '/'，失败返回原串
export function mergeResults(lists: WebSearchResult[][], cap: number): WebSearchResult[]
```

- `mergeResults`：round-robin 交错各源列表（公平呈现两源）→ 按 `normalizeUrl(url)` 去重（首次出现胜）→ 截到 `cap`。
- `cap` 默认常量 `MAX_MERGED_RESULTS = 8`。
- 每条 snippet 截断到 `MAX_SNIPPET_CHARS = 200`（省 token；超出加 `…`）。

## 4. 工具（`src/tools/webSearchTool.ts`）

```ts
const schema = z.object({
  query: z.string().min(2).describe('The search query to use'),
  allowed_domains: z.array(z.string()).optional().describe('Only include search results from these domains'),
  blocked_domains: z.array(z.string()).optional().describe('Never include search results from these domains'),
})

export interface WebSearchConfig { bocha?: string; tavily?: string }  // 各自 apiKey；缺省即不查该源

export function makeWebSearchTool(deps: { config: WebSearchConfig; fetchJson?: FetchJson }): Tool<typeof schema>
```

工具行为（`call`）：
1. **互斥校验**：`allowed_domains` 与 `blocked_domains` 同时非空 → 返回 `错误：不能同时指定 allowed_domains 和 blocked_domains`（对齐 CC）。
2. 组装启用的源：`config.bocha` 有 → bochaSearch；`config.tavily` 有 → tavilySearch。**两者都无 → 返回 `错误：未配置任何搜索源（在 settings.json 的 webSearch 填 bocha/tavily 的 apiKey）`**。
3. `Promise.allSettled` 并发查所有启用源（传 `count:5`、域名过滤、`ctx.signal`）。
4. 收集 fulfilled 的结果列表；**Bocha 列表客户端域名过滤**（allowed→仅留命中域；blocked→剔除命中域；用 host 后缀匹配）；Tavily 已原生过滤不重复过滤。
5. 全部源都 rejected → 返回 `错误：搜索失败（<各源错误摘要>）`。
6. `mergeResults` 合并去重截顶 → 空 → 返回 `未找到结果：<query>`。
7. 格式化输出（§5）。

**flags**（镜像 WebFetch）：`isReadOnly: false`；`needsPermission: () => 'WebSearch'`（**常量**——一条 allow-rule 覆盖所有搜索，对齐 CC `{toolName:'WebSearch'}` 粒度；query 会在审批弹窗的参数预览里显示，无需进规则键。若返回 `WebSearch <query>` 则每个不重复的 query 都要重批，无意义）。

## 5. 输出格式（移植 CC）

```
Web search results for query: "<query>"

1. <title>
   <url>
   <snippet>

2. …

REMINDER: 回答中必须用 markdown 超链接把上面的来源列在末尾的 "Sources:" 段（[标题](URL)）。
```

空结果不带编号列表，直接 `未找到结果：<query>`。

## 6. 配置（`src/config.ts`）

```ts
export interface WebSearchSettings { bocha?: { apiKey?: string }; tavily?: { apiKey?: string }; provider?: string }
// Settings.webSearch?: WebSearchSettings
```

`parseWebSearchConfig(raw): WebSearchSettings | undefined`（宽松解析，镜像 parseMcpServers 风格）：非对象→undefined；bocha/tavily 须为对象且 apiKey 为非空 string 才留；`provider` 留作向后兼容但**不使用**。

**key 来源优先级**（接线层组装 `WebSearchConfig`）：env `BOCHA_API_KEY`/`TAVILY_API_KEY` > `settings.webSearch.{bocha,tavily}.apiKey`。任一源无 key 则该源不参与查询。

loadSettings 加 `webSearch: parseWebSearchConfig(raw?.webSearch)`。

## 9. 记录偏离 / 取舍

1. **搜索后端非服务端**：CC 用 Anthropic server tool，deepcode 客户端直连第三方（机制对齐非实现）。
2. **双源并查并集**（用户钦定）：烧双份额度 + 延迟取最慢家；一源失败用另一源（resilience）。
3. **结果带 snippet**（比 CC 纯 title+url 富）：Tavily/Bocha 直接给摘要、deepcode 无 CC 的 sub-model 总结层，带 snippet 更有用；snippet 截 200 字省 token。
4. **Bocha 客户端域名过滤**（无原生）：保 CC schema 在两源都生效。
5. **必走代理**：undici+ProxyAgent（与 webfetch 一致），否则本机连不通搜索 API。
6. **Bocha caveat**（实查未能从登录墙后官方文档 100% 确认）：freshness 大小写用小写（对齐官方 MCP server）、无公开限流——按 4 份独立代码实现的字段形态实现，落地后真机验证一次。
7. **settings `provider` 字段**忽略（双源并查无需选源；保留不报错向后兼容）。
8. **无 token 计费**：WebSearch 不调 DeepSeek（纯外部 HTTP），不接 onUsage。
