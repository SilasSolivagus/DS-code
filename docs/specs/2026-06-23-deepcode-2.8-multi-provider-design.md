# deepcode 2.8 通用多 provider / 第三方模型切换 — 设计

- 日期：2026-06-23
- 状态：设计已批准（待实现）
- 分支：`feat/2.8-multi-provider`
- 相关：master roadmap 第 2 层 2.8；信任边界沿用 3.9（`docs/specs/2026-06-18-deepcode-3.9-settings-layering-design.md`）

## 1. 目标与范围

让 deepcode 支持除 DeepSeek 外的任意 OpenAI 兼容后端（首批内置 GLM/智谱），并允许「主任务 / subagent 在**同一个 active provider 内**切能力档」。

**已定方向（经两轮 opus 专家评审 + 用户拍板，不再翻）：**

- **方案 A — 同 provider 内切档**：运行时锁定单一 active provider（全 DeepSeek 或全 GLM），主循环与各 subagent 在该 provider 的 fast/smart 档之间切。**不做**跨 provider per-subagent（主 DeepSeek、子 GLM 混跑）——那是 CC 本身都没有的能力，价值边缘、复杂度大。
- **preset 策略**：内置 `deepseek`（默认）+ `glm` 两个 curated preset，外加 `custom` 通用口（用户在 settings 自填 OpenAI 兼容后端）。
- **provider 只在 settings 配**（启动生效），**不加** `/provider` 运行时命令（归 TUI 最后批）。
- **砍 B-ready**：不预留跨厂路由骨架（无 client Map 缓存、无 provider-qualified 元数据 key、无「留口不接线」）。方案 A 已否决跨厂，B-ready 是为已否决场景铺路，违反 YAGNI。真要做方案 B 时再局部改。

**CC 对齐依据（实读 `/Users/silas/Desktop/src` 确认）**：CC 的「某任务用 sonnet、某 subagent 用 opus」本质是**同一个全局 provider（Anthropic）内换 model id 字符串、共用调用路径**；provider 由全局 env 一次性决定，无 per-request 入口；AgentTool 的 model 参数被锁死为 `['sonnet','opus','haiku']` 能力档别名。方案 A 是 CC 模型的直译。

## 2. 已实测的 GLM 事实（curl 实打，2026-06-22/23）

| 项 | 结论 |
|---|---|
| baseURL | `https://open.bigmodel.cn/api/paas/v4`，OpenAI 兼容 `/chat/completions` |
| 模型目录 | `GET /api/paas/v4/models` 返回本账号 8 个当前世代模型：`glm-4.5` `glm-4.5-air` `glm-4.6` `glm-4.7` `glm-5` `glm-5-turbo` `glm-5.1` `glm-5.2` |
| 全目录均 reasoning | 实测 8 个目录模型**全部**返回 `reasoning_content`、全部支持 thinking（与 DeepSeek flash/pro 同理，整代支持） |
| 遗留档 | `glm-4-flash`/`glm-4-flashx` chat 仍可调但**不在目录**（遗留/免费档，非 reasoning，可能被淘汰，preset **不收录**） |
| thinking 字段 | GLM 接受 `thinking:{type:enabled/disabled}` + `reasoning_effort`，**与 DeepSeek 同方言**；非 reasoning 的遗留档容忍并忽略（不报错） |
| usage 缓存 token | GLM 在 `prompt_tokens_details.cached_tokens`（嵌套）；DeepSeek 在顶层 `prompt_cache_hit_tokens` |
| 流式 usage | 需 `stream_options:{include_usage:true}`（deepcode `api.ts:124-125` 已发），GLM 兼容 |
| GLM-5.2 定价 | 国际(z.ai) 输入 $1.40/M、输出 $4.40/M、缓存 $0.26/M；bigmodel.cn 的 CNY 价**待实现期核实** |

**方言差异收敛为三处**（见 §5）：① thinking 三态门控；② usage 缓存字段路径；③ createClient 的 baseURL/apiKeyEnv。其余（消息格式、stream_options、reasoning_content）全共享。

## 3. Provider preset 抽象（新模块 `src/providers.ts`）

```ts
type Dialect = 'deepseek' | 'glm' | 'openai'   // 'openai' 给 custom：不发任何非标字段(thinking/reasoning_effort)

interface ModelMeta {
  hit: number          // 缓存命中输入价 CNY / 1M（镜像 pricing.ts PRICES.hit）
  miss: number         // 未命中输入价 CNY / 1M（PRICES.miss）
  out: number          // 输出价 CNY / 1M（PRICES.out）
  contextWindow: number
  supportsThinking: boolean
}

interface ProviderPreset {
  id: 'deepseek' | 'glm' | 'custom'
  baseURL: string
  apiKeyEnv: string                       // 'DEEPSEEK_API_KEY' / 'ZHIPUAI_API_KEY'
  dialect: Dialect
  cjkRatio: number; otherRatio: number    // preset 级（非 per-model）；默认 0.6 / 0.3
  models: { fast: string; smart: string }
  meta: Record<string, ModelMeta>         // 裸 model id 为 key
}
```

**内置 preset：**

- `deepseek`（默认，行为字节级等价现状）：baseURL `https://api.deepseek.com`、apiKeyEnv `DEEPSEEK_API_KEY`、dialect `deepseek`、cjk 0.6/0.3、models `{fast:'deepseek-v4-flash', smart:'deepseek-v4-pro'}`、meta 复用 `pricing.ts` + `tokenEstimate.ts` 现有数字（flash `{hit:0.02,miss:1,out:2,window:1e6,thinking:true}`、pro `{hit:0.025,miss:3,out:6,window:1e6,thinking:true}`）。
- `glm`：baseURL `https://open.bigmodel.cn/api/paas/v4`、apiKeyEnv `ZHIPUAI_API_KEY`、dialect `glm`、cjk 0.6/0.3、models `{fast:'glm-5-turbo', smart:'glm-5.2'}`、meta **收录全部 8 个目录模型**（`supportsThinking:true` 全开，便于用户 `/model <任意目录 id>` 直接用且计价/window 正确）：
  - `glm-5.2` `{window:1_000_000, thinking:true}`、`glm-5.1`/`glm-5`/`glm-5-turbo` `{window:200_000(待核实), thinking:true}`
  - `glm-4.7`/`glm-4.6` `{window:200_000, thinking:true}`、`glm-4.5`/`glm-4.5-air` `{window:128_000(待核实), thinking:true}`
  - 各档 `hit/miss/out` CNY 单价**待实现期核实**（`bigmodel.cn/pricing`）；fast=`glm-5-turbo`、smart=`glm-5.2`
- `custom`：从 `settings.providers.custom` 读全部字段（dialect 默认 `openai`）。

**active preset 解析**：`resolveActiveProvider(settings) -> ProviderPreset`。`settings.provider`（默认 `'deepseek'`）选内置；`'custom'` 时与 `settings.providers.custom` 合并。新增辅助：`activeFastModel()`、`activeSmartModel()`、`activeModelMeta(modelId)`（先查 active preset.meta 裸 id，未知→fail-safe 默认）。

## 4. 配置形态 + 信任边界（**双动作，缺一不可**）

`config.ts` Settings 新增：

```ts
provider?: 'deepseek' | 'glm' | 'custom'          // 默认 'deepseek'
providers?: {
  glm?:    { apiKey?: string }                    // 仅覆盖 key（preset 其余内置）
  custom?: { baseURL: string; apiKeyEnv?: string; apiKey?: string;
             dialect?: Dialect; models: {fast:string;smart:string}; meta?: Record<string,ModelMeta> }
}
```

apiKey 来源优先级：`env[preset.apiKeyEnv]` → `settings.providers.<id>.apiKey`（文件 600）。用户的 GLM key 已存 `~/.deepcode/settings.json` 的 `providers.glm.apiKey`（600，不进仓库）。

**信任边界（settingsLayers.ts）—— 3.9 是显式白名单机制，新字段不会自动安全：**

1. **`parsePresent`（`settingsLayers.ts:146` copy 列表）必须注册 `provider` + `providers`** —— 否则分层加载器根本不读这俩字段，**连 user scope 配的都不生效、功能完全不工作**。
2. **`DANGEROUS_TOP_KEYS`（`settingsLayers.ts:14`）必须加 `provider` + `providers`** —— 否则一旦在 parsePresent 注册，project / git-tracked-local scope 就能注入 `providers.custom.baseURL` + `apiKey`，把流量导去攻击者端点 + 偷 key，**绕过 3.9 对 baseURL/apiKey 的剥离**。

即：provider 路由配置只信任 user scope，project scope 一律剥离。

## 5. 方言 adapter（`api.ts` 按 `preset.dialect` 分叉，三处）

1. **thinking 三态门控**（修 `api.ts:127-129` 当前二选一）：
   - `supportsThinking=true` 且本轮 thinking 开 → `{reasoning_effort, thinking:{type:'enabled'}}`
   - `supportsThinking=true` 且本轮 thinking 关 → `{thinking:{type:'disabled'}}`
   - `supportsThinking=false`（dialect=`openai` 的 custom 端点、或 GLM 遗留非 reasoning 档）→ **整个 thinking + reasoning_effort 字段都不发**（GLM 目录全支持 thinking，但 custom 第三方端点可能拒绝非标字段，三态门控主要为 custom/openai 方言防御）。
2. **usage 缓存字段归一**（**归一点定死在 `Assembler.push`，`api.ts:30-36`，不在消费端分叉**）：
   - dialect `deepseek` → 读顶层 `chunk.usage.prompt_cache_hit_tokens`
   - dialect `glm` → 读 `chunk.usage.prompt_tokens_details?.cached_tokens`
   - dialect `openai` → 缺省 0
   - 统一归一成 `Usage.prompt_cache_hit_tokens`，下游（`session.ts:15`、`stats.ts:41`、`headless.ts:67`、`commands.ts:50`、`useChat.ts:148/315`）**零改动**。
3. **createClient 读 preset**（修 `api.ts:90-95,88`）：baseURL ← `preset.baseURL`、apiKey ← `env[preset.apiKeyEnv] ?? settings.providers.<id>.apiKey`；key 缺失时错误文案带 provider 名 + 正确 env 变量名。

## 6. 干掉硬编码（**grep 兜底全扫，不只改 SUB_MODEL 常量**）

SUB_MODEL 常量重构只覆盖 import 点；**裸字符串 `'deepseek-v4-flash'`/`'deepseek-v4-pro'` 不在引用链上，必须 grep 兜底**，否则 GLM 用户在常见路径静默打错端点：

| 位置 | 类型 | 改法 |
|---|---|---|
| `tools/constants.ts:3` `SUB_MODEL` | 常量 | → `activeFastModel()` 函数（解析 active preset.fast） |
| `tools/agent.ts:49`、`tools/webfetch.ts:70`、`tools/skill.ts:47`、`hookRuntime.ts:22` | import 消费 | → 调 `activeFastModel()` |
| `compact.ts:19` | **裸字符串** | → `activeFastModel()`（compact 总结模型，高频，漏了每次压缩 404） |
| `useChat.ts:519-520` | **裸字符串** | → active fast（compact usage 标签） |
| `useChat.ts:229` | 启动默认 model | → `settings.model ?? activeFastModel()` |
| `useChat.ts:844` | `/model` 轮换 | → active `fast↔smart` 轮换 |
| `useChat.ts:838-839` | `/model` 计价提示 | → 按 active provider（删「非 deepseek 计价 0」死文案） |
| `session.ts:90,103` | resume 默认 | → active fast（见 §7） |
| `headless.ts:42` | 硬编码 model | → `settings.model ?? activeFastModel()`（见 §8） |
| `pricing.ts` `costCNY`/`cacheSavingsCNY` | 查 `PRICES[model]` | → 查 `activeModelMeta(model)`，未知 fail-safe 0 |
| `tokenEstimate.ts:46-49,57` `resolveContextWindow` | 查 `MODEL_CONTEXT_WINDOWS` | → 查 `activeModelMeta(model).contextWindow`，未知→默认 200k |
| `tokenEstimate.ts:18` CJK 比例 | 硬编码 0.6/0.3 | → active preset `cjkRatio`/`otherRatio`（preset 级注入，纯函数加参） |

**不做 provider-qualified key**：元数据查表用「active preset.meta 的裸 id」即可——现有 deepseek 用户裸 id 在 deepseek meta 命中（零破坏）、GLM 用户裸 id 在 glm meta 命中。**向后兼容天然成立**，无需 qualified key 那层。

## 7. 档语义映射 + 具体 id 透传（`agentsLoader.ts:37-45`）

`resolveAgentModelAlias` 改为（取代硬编码 `startsWith('deepseek')` 白名单）：

- `inherit` / 空 → 父模型
- `opus` / `sonnet` / `best` / `smart` → active preset.smart
- `haiku` / `flash` / `fast` → active preset.fast
- **active preset 命名空间内的合法具体 id（在 `preset.meta` 里）→ 直接透传**（让 GLM 用户能在 `.claude/agents` 写 `model: glm-4.6` / `glm-4-flashx` 等中间档，不被静默吞）
- 其余（跨 provider id / 不认识）→ `inherit`（保留安全兜底，绝不把跑不了的 id 喂 API）

subagent 仍共用同一个 active client（方案 A）。

## 8. resume / session / headless（**必修，草案曾遗漏**）

- **session meta 存 `providerId`**（`session.ts` 落盘加字段）。resume 时：若 session 记录的 model **不属于 active provider**（不在 `activeModelMeta` 命中且不等于 active fast/smart）→ 回落 active fast（或保留 model 但用 active client 时给出一次性提示）。避免「用 deepseek 端点请求 glm-5.2」的端点漂移报错。
- **headless（`headless.ts:42`）** 读 `settings.model ?? activeFastModel()`，不再硬编码，使 `-p` 非交互模式遵循 active provider。

## 9. 不做（YAGNI）

- 跨厂 per-subagent（方案 B 本体）、B-ready 骨架（client Map / qualified key / 留口）。
- `/provider` 运行时命令、状态栏显示 provider 名（归 TUI 最后批）。
- per-model cjkRatio（降 preset 级）、多模态、GLM Coding Plan 订阅计费、SSRF/baseURL 白名单（主 API client 不走 ssrfGuard，无需加白名单；安全点在 `providers.custom.baseURL` 的 scope 剥离，见 §4）。

## 10. 验收标准（每条配断言测试）

**必修（不过不能合并）：**

1. `provider:'glm'` 时，`resolveActiveProvider` 返回 GLM preset，createClient 用 `open.bigmodel.cn` baseURL + `ZHIPUAI_API_KEY`/settings key。
2. `settingsLayers`：① user scope 配 `provider`/`providers` 能被 `loadSettings` 读出生效；② project scope 注入 `providers.custom.baseURL`+`apiKey` 被剥离（断言生效配置里不含 project 的恶意值）。
3. compact 路径（`compact.ts`）在 `provider:'glm'` 下用 `glm-5-turbo`（active fast），**不打 `deepseek-v4-flash`**。
4. headless `-p` 在 `provider:'glm'` 下用 active provider，不硬编码 deepseek。
5. resume 一个 model=`glm-5.2` 的 session 而 active=deepseek 时，不产生端点漂移（回落或提示，断言不抛）。
6. thinking 三态：`supportsThinking:false` 的模型请求体**不含** thinking/reasoning_effort 字段；`true` 模型按开关发 enabled/disabled。
7. usage 归一：glm dialect 下 `Assembler` 把 `prompt_tokens_details.cached_tokens` 归一进 `prompt_cache_hit_tokens`，下游成本/缓存率计算正确。

**应修：**

8. `costCNY`/`resolveContextWindow` 对 GLM 模型走 active meta（非恒 0 / 非恒默认），未知模型仍 fail-safe。
9. `agentsLoader` 在 active=glm 下：`model:glm-4.6` 透传、`model:opus` → glm-5.2、`model:deepseek-v4-pro`（跨 provider）→ inherit。

**向后兼容（必须绿）：**

10. 不配 `provider`（或配 deepseek）时，全链路行为与现状字节级等价；现有裸 id `deepseek-v4-flash` 在所有查表点命中。

## 11. 风险与 follow-up

- 8 个 GLM 目录模型的 **CNY 精确单价 + 各档 context window 待实现期核实**（`bigmodel.cn/pricing` JS 渲染，需登录或查文档；仅 glm-5.2=1M 已确认）；未核实前 meta 填国际价换算估值并标注，`DEEPCODE_MAX_CONTEXT_TOKENS` env 可覆盖 window。
- `custom` dialect 默认 `openai`（不发非标字段）；若某第三方端点需要 thinking，用户可在 `providers.custom.dialect` 显式设 `glm`/`deepseek`。
- 状态栏不显 provider 名属本期不做；用户切 provider 后靠 `/model` 提示与成本判断当前后端，TUI 最后批补。
