# deepcode 2.1 · Token Budget 自动续跑 — 设计 spec

**日期：** 2026-06-22
**roadmap：** 第 2 层 2.1（adaptive thinking 对 DeepSeek N/A，真实剩余=CC 的输出 token budget 自动续跑）
**CC 对照：** `utils/tokenBudget.ts`（解析）+ `query/tokenBudget.ts`（续跑决策）+ `bootstrap/state.ts`（sticky 状态）+ `REPL.tsx:2893`（入口）

## 1. 目标 / 动机

让用户对大任务说「别提前停，肯花 token，干到完」。用户在输入里写 `+500k` / `use 2M tokens` 设一个**输出 token 目标**；本轮 agentic 循环只要累计输出 token 未达目标×90% 且仍有进展，就自动注入「keep working」nudge 续跑（受 maxTurns 约束），到目标或收益递减就停。

**N/A（不做）：** CC 的 adaptive thinking / budgetTokens（Anthropic 专有 API），deepcode 已有 effort 三档+ultrathink 关键词等价。CC 的 API task_budget beta（Anthropic header）。

## 2. 用户决策（2026-06-22 brainstorm）

- **D1 预算生命期 = sticky（对齐 CC）**：`+500k` 设一次后**持久**，每条后续 send 都沼用该目标（每条 send 的输出计数各自归零，独立冲目标），直到用户改/清。
- **D2 范围 = 含状态栏显示（碰 TUI）**：状态栏加「budget 已用/目标」段；按 TUI 冒烟约定（Claude 备命令、用户跑+截图）。

## 3. 设计

### 3.1 解析（纯函数，新模块 `src/tokenBudget.ts`）
`parseTokenBudget(input: string): number | null`，**直接港 CC 三正则**（强制 k/m/b 后缀，避免 `+5` 等自然语言误匹配）：
- `SHORTHAND_START = /^\s*\+(\d+(?:\.\d+)?)\s*(k|m|b)\b/i`（开头 `+500k`）
- `SHORTHAND_END = /\s\+(\d+(?:\.\d+)?)\s*(k|m|b)\s*[.!?]?\s*$/i`（结尾 ` +500k.`）
- `VERBOSE = /\b(?:use|spend)\s+(\d+(?:\.\d+)?)\s*(k|m|b)\s*tokens?\b/i`（`use 2M tokens`）
- 乘数 k=1e3 / m=1e6 / b=1e9；`parseFloat(值)*乘数`。
- **必须带后缀**：故 `+1000000`（无后缀）不匹配、`+5`（自然语言）不匹配。**清除**预算用 `+0k`（→0，见 3.3）。
- 无匹配 → `null`（不含预算指令；sticky 下=沿用上次）。
- 只解析、不剥离输入文本（CC 也保留原文给模型；findTokenBudgetPositions 高亮不做）。

### 3.2 续跑决策（纯函数，`src/tokenBudget.ts`）
`shouldContinueForBudget(state): boolean`，对齐 CC `query/tokenBudget.ts`：
- 入参：`{ budget, outputSoFar, continuations, lastDeltas: number[] }`。
- `COMPLETION_THRESHOLD = 0.9`：`outputSoFar < budget * 0.9` 才考虑续跑。
- **收益递减熔断 `DIMINISHING_THRESHOLD = 500`**：`continuations >= 3` 且最近两次 delta 都 `< 500` → 返回 false（**sticky 刚需**：防设了 +500k 后回「hi」狂烧）。
- `budget <= 0` 或 null → false（无预算/已清）。
- 子代理不参与（在 runLoop 调用点用 `deps.isSubagent`/agentId gate，子循环不传 budget）。

### 3.3 sticky 状态（useChat 层，进程内存，不落 session）
- `currentTokenBudget: number | null`，初始 null。
- 每条 send 前：`const parsed = parseTokenBudget(input); if (parsed !== null) currentTokenBudget = parsed === 0 ? null : parsed`（`+0` 清除；显式新值覆盖；无指令沿用）。
- 不持久化到 session/磁盘（对齐 CC 的进程内 module 状态，重启即清；避免 resume 后悄悄烧钱）。
- 清除途径：`+0` / `use 0 tokens`，或后续可加 `/budget` 命令（本期只做 `+0`）。

### 3.4 接入 runLoop（`src/loop.ts`）
- `LoopDeps` 加 `tokenBudget?: number`（caller 传当前 sticky 值）。
- runLoop 内累计本次 send 的输出 token：`outputSoFar += result.usage.completion_tokens`（每轮 turn_end 后）；记录每轮 delta 入 `lastDeltas`。
- **新续跑分支**：在 `if (!result.toolCalls.length)` 自然结束块内、length 截断分支**之后**、任务通知分支**之前**加：
  ```
  if (deps.tokenBudget && shouldContinueForBudget({budget, outputSoFar, continuations, lastDeltas})) {
    continuations++
    messages.push({ role:'user', content: '（继续——尚未达到本次 token 预算，请接着完成未尽工作，不要总结收尾。）' })
    continue
  }
  ```
- **与 length 截断续跑统一**：length 截断分支保持优先（截断必须续写）；budget 续跑是「没截断、模型想停但预算没到」的补充。两者都 `continue` 进下一轮、都受 maxTurns 约束，不会双重注入（length 分支 `continue` 后不会落到 budget 分支）。
- 受 `maxTurns ?? 80` 硬约束（budget 续跑也在 for 循环内，天然封顶）。

### 3.5 状态栏显示（`StatusFooter.tsx`，碰 TUI）
- 新增可选 props `tokenBudget?: number`、`budgetUsed?: number`。
- 仅 `tokenBudget` 有值时，在 Row 2（Context 行）尾部加一段：`· budget {fmtK(used)}/{fmtK(budget)}`，颜色复用 dim（达标接近时可后续加色，本期 dim）。
- useChat 暴露 `tokenBudget()` + 本条 send 的 `budgetUsed()`（累计输出）；App/FullscreenApp 接线。

## 4. 测试（TDD）

**纯函数（tokenBudget.test.ts）：**
- parseTokenBudget：`+500k`→500000 / `+2m`→2e6 / `use 500k tokens`→500000 / `+1000000`→1e6 / 大小写 / `+0`→0 / 无匹配→null / 普通文本不误匹配。
- shouldContinueForBudget：未达 90%→true / 达 90%→false / null/0→false / continuations≥3 且最近两 delta<500→false（熔断）/ delta 仍大→继续 true。

**loop 集成（loop.test.ts 扩展）：**
- 设 budget、模型无 toolCall 自然结束但 outputSoFar<90% → 注入续跑 nudge 进下一轮。
- 达标 → 不续跑、return done。
- 熔断：连续 3 轮小 delta → 停。
- 无 budget → 行为不变（回归）。
- length 截断 + budget 同时 → length 优先、不双重注入。

**TUI（statusfooter.test.tsx 扩展）：** budget 有值显 `budget X/Y`；无值不显。

## 5. 砍 / 偏离（记录）
- 不做输入文本高亮（CC findThinkingTriggerPositions 类，UI 装饰）。
- 不持久化 budget 到 session（进程内，重启清；对齐 CC module 状态、且防 resume 悄悄烧钱）。
- 清除只做 `+0`，`/budget` 命令留后续。
- 状态栏 budget 段本期 dim 不变色（达标变色可后续）。
- headless：本期入口同 useChat（若 headless 也接受用户输入则同样解析）；最小可只做 TUI 路径。
