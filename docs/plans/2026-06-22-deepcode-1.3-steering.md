# 1.3 Steering（用户中途转向）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让用户在模型跑当前 turn 时插入消息——Enter 排队到 tool_result 边界注入、Alt+Enter 软中断立即重定向——对齐 CC steering（保留进度、非硬中断）。

**Architecture:** 新增会话级 FIFO `SteeringQueue`（`src/steering.ts`）。loop 在 tool_result 边界 drain 队列注入 user 消息（复用 drainInjections 同站点）；`now` 经 `abort.abort('interrupt')` 软中断，loop 按原生 `signal.reason` 区分 interrupt（保留进度+重建 signal+注入续跑）/ user-cancel（现有硬中断）。TUI 输入框 busy 时 Enter→enqueue、Alt+Enter→enqueue+interrupt、ESC→有队列拉回否则硬中断。

**Tech Stack:** TypeScript/ESM、ink5 TUI、vitest、OpenAI SDK 消息形状。

## Global Constraints

- 测试框架 vitest，测试在 `test/*.test.ts`；运行 `npx vitest run test/<file>`，全量 `npm test`，类型 `npm run typecheck`，构建 `npm run build`。
- **禁用** `Date.now()` / `Math.random()` 于构造期不可控位置——id 用递增计数器 + 前缀（同 `taskList.ts` 风格）。
- 注入格式固定：`<queued-user-message>\n用户在你执行过程中补充了这条消息（在你看到它之前已发出）。请据此调整当前工作：\n{value}\n</queued-user-message>`。
- abort reason 复用原生 `AbortSignal.reason`：steering=`'interrupt'`、ESC=`'user-cancel'`。
- headless 不接 steering（无交互输入源），仅 TUI。
- steering 纯内存、不落盘、不跨 resume。
- 每个写工具/新增导出务必保持现有测试计数断言同步（若有）。

```ts
// src/steering.ts —— 会话级 steering 队列：用户中途插入消息的 FIFO 缓冲。
// priority 仅用于 TUI 列队展示标记 + 决定 onSteerNow 是否触发 abort（不影响 drain）；
// deepcode 无 Sleep 工具，无 later-only 边界，故 drain 一律 drainAll（对齐 spec §6「later 暂同 next」）。

export type SteeringPriority = 'now' | 'next' | 'later'

export interface SteeringItem {
  id: string
  value: string
  priority: SteeringPriority
}

export function formatSteeringMessage(value: string): string {
  return `<queued-user-message>\n用户在你执行过程中补充了这条消息（在你看到它之前已发出）。请据此调整当前工作：\n${value}\n</queued-user-message>`
}

export class SteeringQueue {
  #items: SteeringItem[] = []
  #nextId = 1
  #subs = new Set<() => void>()

  get size(): number { return this.#items.length }

  enqueue(value: string, priority: SteeringPriority): SteeringItem {
    const item: SteeringItem = { id: `steer-${this.#nextId++}`, value, priority }
    this.#items.push(item)
    this.#notify()
    return item
  }

  drainAll(): SteeringItem[] {
    if (this.#items.length === 0) return []
    const out = this.#items
    this.#items = []
    this.#notify()
    return out
  }

  popLast(): SteeringItem | undefined {
    const item = this.#items.pop()
    if (item) this.#notify()
    return item
  }

  peek(): readonly SteeringItem[] { return this.#items }

  clear(): void {
    if (this.#items.length === 0) return
    this.#items = []
    this.#notify()
  }

  subscribe(fn: () => void): () => void {
    this.#subs.add(fn)
    return () => { this.#subs.delete(fn) }
  }

  #notify(): void { for (const fn of this.#subs) fn() }
}
```
