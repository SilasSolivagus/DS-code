# 2.4 Prompt 缓存对齐 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 DeepSeek 自动前缀缓存的命中率与节省金额上到状态栏，并把「前缀稳定」纪律固化成回归守卫测试。

**Architecture:** 三块真实增量 + 一块守卫测试。计费/命中率/缓存友好 prompt 已存在（见 spec 现状盘点），本计划只新增：`cacheSavingsUSD` 纯函数（pricing.ts）→ `createChatCore` 暴露 `cacheSavings()` → `StatusFooter` Row 2 渲染 `cache N% (−$X)` → 前缀稳定性守卫测试。

**Tech Stack:** TypeScript/ESM、vitest、ink5（TUI）、ink-testing-library。

## Global Constraints

- DeepSeek 自动前缀缓存，**不引入** CC 的 `cache_control`/TTL/scope/断点/break 检测/缓存开关。
- 不读 `prompt_cache_miss_tokens`（推导 `miss = prompt − hit` 已正确）。
- 每任务结束前该任务相关测试 + `npm run -s typecheck` 通过；全部任务完成后全量 `npm test` + typecheck + build 全绿。
- 组件 3 碰 TUI → 合 main 前用户真机 `npm start` 冒烟。
- 现有 ESM import 必须带 `.js` 后缀（项目惯例）。

```typescript
/**
 * 缓存命中省下的美元金额：命中的 hitTokens 若按未命中价计本要多花的钱。
 * = hitTokens × (miss − hit) / 1e6。未知模型返回 0。
 */
export function cacheSavingsUSD(model: string, hitTokens: number): number {
  const p = PRICES[model]
  if (!p) return 0
  return (hitTokens * (p.miss - p.hit)) / 1_000_000
}
```

  it('reminder 注入不修改已存在的前缀消息（缓存守卫）', async () => {
    script.push(
      {
        result: {
          content: '', toolCalls: [{ id: 'g1', name: 'Glob', args: '{"pattern":"*"}' }],
          usage, finishReason: 'tool_calls',
        },
      },
      { result: { content: 'ok', toolCalls: [], usage, finishReason: 'stop' } },
    )
    const { globTool } = await import('../src/tools/glob.js')
    const deps = makeDeps([globTool])
    deps.reminders = () => ['前缀守卫提醒']
    const messages: any[] = [
      { role: 'system', content: 'SYS-PREFIX' },
      { role: 'user', content: 'hi' },
    ]
    await drain(runLoop(messages, deps))
    // 前缀（system / user）字节不变——reminder 不回头改前缀，DeepSeek 自动缓存才命中
    expect(messages[0]).toEqual({ role: 'system', content: 'SYS-PREFIX' })

    // reminder 只落在最后一条 tool 消息
    const toolMsgs = messages.filter(m => m.role === 'tool')
    expect(toolMsgs[toolMsgs.length - 1].content).toContain('前缀守卫提醒')
  })
